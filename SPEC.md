# AI保顾问功能 SPEC v0.1

## Problem Statement

保险销售顾问手里有客户生日表、保单续期/缴费表，以及日常自己新增的客户跟进待办。现在这些提醒分散在 Excel、个人记忆、日历或聊天工具里，容易出现以下问题：

- 客户生日、保单续期、缴费到期、个人待办无法统一查看。
- 保单续期需要根据生效时间、缴费年期、已过缴费周年等信息计算，人工判断容易漏。
- 多张表里的同一客户可能因证件号展示方式不同而难以识别。
- 提醒结果需要能同步到外部工具，方便后续查看和处理。

第一版目标是做一个单个保险销售顾问可本地运行的客户工作日历，支持导入 Excel、生成生日/续期/手动待办提醒、维护完成状态，并提供本地飞书同步能力。

## Proposed Solution

第一版采用本地网页 + Node.js/TypeScript + SQLite + 飞书 CLI 同步方法。

本地网页作为主要操作入口。当前工作台以月历为主，右侧显示选中日期的具体提醒。页面不应重复展示日历已有的年月、上月、下月、今天等控件；低频统计、数据健康、表结构、同步细节不应占据日常工作主视觉。

生日、续期、手动待办使用颜色区分。右侧详情卡片不重复写分类标签，生日卡片展示客户姓名，续期卡片展示客户姓名和保费，点开后再展示对应保单详情。完成按钮必须写入本地状态；选中日期支持一键完成当天全部未完成提醒。

开发体验原则：

- 如无必要，不新增实体按钮、导航入口或常驻控件。
- 若必须新增入口，必须说明它如何改善客户体验，或如何帮助保险从业者开展业务。
- 功能应尽量独立，避免后续修改时牵连无关流程。
- 高频工作界面只保留高频动作，低频配置和调试信息不应占据主界面。

## Reminder Groups

- 客户生日提醒：导入生日表，根据客户生日计算今年或下一次生日。如果今年生日已过，则生成下一年生日提醒。
- 保单续期提醒：导入保单续期表，根据明确字段判断生效时间、缴费期间、缴费日期等信息。只相信明确缴费期间字段，不从产品名推断。只生成未来 60 天内真正需要跟进的续期提醒；更远的续期作为保单计划信息展示下次续期和缴费结束年。若缴费期已结束，不生成续期提醒，标记为缴费期已结束。
- 手动待办提醒：用户可在本地网页新增待办，至少包含标题、日期、可选关联客户。状态只需要“未处理 / 已完成”。

## Identity Matching

- 优先使用姓名 + 证件号匹配。
- 若一个表是完整证件号，另一个表是前后位脱敏证件号，则必须姓名相同，并且证件号前后位都能严格匹配，才自动识别为同一客户。
- 若姓名不同、证件号前后位无法同时匹配、或业务字段缺失导致无法稳定识别，则不自动合并。
- 不确定的身份匹配进入待确认。

## Renewal Calculation

- 缴费期间只从明确字段读取，例如“缴费期间/交费期间/缴费年限”等。
- 支持类似 `1年`、`3年`、`5年`、`10年`、`20年`、`30年` 的缴费期间。
- 支持类似 `60周岁`、`65周岁`、`70周岁` 的年龄型缴费期间，前提是能取得被保人生日和保单生效日。
- 年龄型缴费期间的截止缴费年按生日与投保日先后判断：若客户当年先过生日后到投保日，`N周岁` 的最后缴费日为客户 `N-1` 岁生日那一年的投保日；若当年先到投保日后过生日，则最后缴费日为客户 `N` 岁生日那一年的投保日。同一天按生日已到处理。
- 续期日优先使用保单生效时间的月日。
- 已缴年份按生效时间到提醒年份的保单周年计算，不依赖样例表中的“保单年度/期数”。
- 只生成未来 60 天内的续期提醒。
- 60 天窗口外的有效保单不生成提醒，但保单详情和飞书保单表应保留下次续期日和缴费期结束年。
- 若计算得出缴费期已经结束，则不生成续期提醒，标记为“缴费期已结束”。

## Feishu Sync

- 用户准备一个可编辑的飞书多维表格链接，并在本机完成 `lark-cli` 授权。
- 同步前先生成计划，用户确认后再执行写入。
- 同步目标是飞书多维表格里的客户表、保单表、提醒表三张表。
- 保单表包含下次续期日和缴费结束年，提醒表只承接本地已生成的行动提醒。
- 提醒同步到飞书多维表格，并创建一个按提醒日期展示的日历视图；续期提醒只包含未来 60 天内需要跟进的保单。
- 用户手动标记为“关键”的提醒，可以额外同步到飞书日历。
- 网页里可以辅助生成计划和触发同步，但仍依赖本机 `lark-cli` 授权；面向普通用户的一键授权不属于当前版本。

## Technical Constraints

- 第一版技术栈为 Node.js / TypeScript。
- 本地数据存储使用 SQLite。
- 飞书集成通过飞书 CLI 完成，不要求当前版本内置完整 OAuth 后台。
- 数据长期保存在本地 SQLite，飞书多维表格作为同步出口和外部看板。
- 客户外部 ID 使用 hash 形态，不包含姓名或证件号。
- 业务字段缺失时允许创建但不自动合并，避免误合并。
- 导入采用增量更新。
- 若发现已有客户/保单/提醒的日期、保费、缴费年期等关键字段发生变化，不直接覆盖，进入待确认变更清单，由用户确认后更新。
- 本地数据库保存必要的完整手机号/证件号用于匹配；网页详情默认脱敏展示。
- 飞书同步时默认脱敏展示手机号和证件号。
- 本地网页支持通过环境变量开启访问密码；默认不开启，便于单机快速使用。
- 第一版 SQLite 文件本身不加密，需要在产品说明中明确本地数据安全风险。
- 同步和日志输出不得展示完整手机号、完整证件号、完整 Base 标识、session cookie 或其他敏感信息。

## Source File Shape

开源仓库不附带真实客户 Excel。导入字段说明见 `docs/sample-data.md`。

当前导入器主要识别两类工作簿：

- 客户信息工作簿：可作为客户生日底册，包含客户姓名、证件号、出生日期、手机号等字段。
- 保单续期/业绩明细工作簿：可作为保单续期底册，包含投保人、被保人、保单号码、保险产品、保费、缴费期间、生效时间等字段。

## Non-goals

- 当前版本不做多人协作、账号体系、权限管理。
- 当前版本不做复杂缴费年期推断，不从产品名、产品规则库、历史经验里猜缴费年期。
- 当前版本不保存完整 Excel 原始行和全部字段，不做完整历史追溯。
- 当前版本不做通用 Excel 字段映射系统。
- 当前版本不自动合并身份信息不足或匹配不严格的客户/保单，宁可产生独立记录，也不做高风险合并。

## Success Criteria

当前版本应满足：

- 能在本地启动网页工具，单个销售顾问可使用。
- 能导入客户生日表和保单续期表，并自动生成生日提醒、续期提醒两个分组。
- 能手动新增客户相关待办，并显示在提醒清单中。
- 能在工作日历中查看当前月份提醒，并查看某一天的具体事项。
- 能根据明确字段计算保单续期计划，只为未来 60 天内的续期生成提醒，并在保单详情保留下次续期日和缴费结束年。
- 能识别缴费期已经结束的保单，并且不再生成续期提醒。
- 能计算年龄型缴费期间；客户生日缺失、关键字段缺失、关键字段变化放入待确认清单。
- 能识别严格匹配的同一客户，尤其支持“完整证件号”与“前后位脱敏证件号”的严格匹配。
- 能进行增量导入；关键字段变化进入待确认清单，不自动覆盖。
- 能生成飞书同步计划，并在用户确认后通过本机 `lark-cli` 将客户表、保单表、提醒表同步到飞书多维表格。
- 能在飞书多维表格中创建提醒日历视图。
- 能将用户手动标记为关键的提醒同步到飞书日历。
