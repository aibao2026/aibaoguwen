# 大规模测试清单

目标：在导入真实客户数据、同步飞书多维表格和日常使用前，逐项确认核心链路可恢复、可复测、可解释。

## 1. 基线冻结

- [ ] 记录当前 Git commit。
- [ ] 执行 `npm run typecheck && npm test && npm run build`。
- [ ] 记录本地统计：客户数、保单数、提醒数、待确认数。
- [ ] 确认 `data/customer-reminders.sqlite` 已备份。

## 2. 导入预检

- [ ] 仅上传客户生日表，点击“预检导入”，确认正式统计不变化。
- [ ] 仅上传保单续期表，点击“预检导入”，确认正式统计不变化。
- [ ] 同时上传两张表，确认预检展示新增客户、保单、提醒、待确认。
- [ ] 用错误文件类型导入，确认系统拒绝而不是写库。
- [ ] 用空表或缺关键列表格测试，确认错误可读。

## 3. 正式导入

- [ ] 正式导入前自动生成备份。
- [ ] 导入后客户、保单、提醒数量符合预期。
- [ ] 重复导入同一批文件，不应无限重复生成客户、保单或提醒。
- [ ] 待确认事项能说明缺生日、缺生效日、缺缴费期间或身份匹配失败。
- [ ] 修正待确认后，对应生日或续期提醒能重新生成。

## 4. 数据恢复

- [ ] 手动创建备份。
- [ ] 新增一条手动待办后恢复备份，确认该待办消失。
- [ ] 恢复后工作台统计刷新。
- [ ] 尝试恢复非备份目录文件，接口应拒绝。

## 5. 客户匹配与隐私

- [ ] 全量证件号表与脱敏证件号表能按姓名和证件号首尾匹配。
- [ ] 匹配不确定时进入待确认，不自动合并。
- [ ] 页面详情只显示脱敏证件号和脱敏手机号。
- [ ] 飞书同步字段不包含完整证件号。

## 6. 续期提醒

- [ ] `10年`、`20年` 等按固定缴费年限计算下次续期和缴费结束年。
- [ ] `60周岁` 等按生日和生效日计算。
- [ ] 只为未来 60 天内的续期生成提醒，60 天外保单不进入提醒表。
- [ ] 已超过缴费期的保单不再生成未来续期。
- [ ] 生效日缺失、缴费期间无法解析时进入待确认。

## 7. 日历与待办

- [ ] 电脑、Pad、手机分别打开工作台。
- [ ] 生日、续期、待办颜色区分清晰。
- [ ] 分组筛选能即时影响日历和右侧详情。
- [ ] 手机端不堆叠低价值统计，只保留核心日历和任务。
- [ ] 在某一天新增个人待办后，日历和提醒列表同步刷新。
- [ ] 完成、恢复、编辑、删除手动待办均可用。

## 8. 飞书多维表格

- [ ] 先执行表结构计划，确认客户、保单、提醒三张表。
- [ ] 执行表结构准备后，飞书里字段完整，保单表包含“下次续期”和“缴费结束年”。
- [ ] 执行日历视图计划，确认目标提醒表和视图名正确。
- [ ] 执行日历视图准备后，飞书里能看到日历视图。
- [ ] 使用“批量创建”计划，确认批次数和记录数合理。
- [ ] 确认飞书提醒表里的续期只包含未来 60 天内需要行动的事项。
- [ ] 执行“批量创建”后，客户、保单、提醒写入成功。
- [ ] 再次执行“批量创建”计划，应跳过已同步记录。
- [ ] 少量修补时使用“增量更新”，确认已有记录可更新。

## 9. 飞书日历

- [ ] 关键提醒计划只包含 `isKey=true` 且未完成的提醒。
- [ ] 限制条数时，只同步指定数量。
- [ ] 全量执行必须勾选“允许全量执行”。
- [ ] 重复执行不会重复创建已同步日程。

## 10. 性能与异常

- [ ] 1000 条提醒内页面滚动不卡顿。
- [ ] 5000 条提醒下提醒列表搜索仍可接受。
- [ ] 飞书 CLI 未登录时，错误提示可读。
- [ ] 网络失败或 CLI 失败时，本地数据不丢失。
- [ ] 大规模测试后可通过备份恢复到测试前状态。

## 11. 发布前开源检查

- [ ] `git status --short --ignored` 中确认 `data/`、`.env`、真实 Excel 和本地备份没有被追踪。
- [ ] 搜索完整手机号、完整证件号、Base token、session cookie，确认没有出现在公开文档、测试快照或日志样例里。
- [ ] 确认 README、SPEC 和 docs 中只描述当前版本已经实现或明确标注为当前不包含的能力。
- [ ] 确认公开图片资源不包含私人二维码、内部品牌素材、客户截图或聊天截图。
