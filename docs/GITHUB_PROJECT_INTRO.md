# AI保顾问：给保险人的本地客户工作日历

AI保顾问是一个面向保险销售顾问、保险经纪人和个人保险从业者的开源客户工作日历。它不是传统后台管理系统，也不是复杂的保险 CRM，而是一个可以在本机运行的提醒和待办工具。

它解决的核心问题很简单：每天打开电脑后，保险人应该马上知道今天该联系谁、为什么联系、涉及哪张保单、保费是多少、这件事有没有真正完成。

## 为什么做这个项目

很多保险从业者的客户信息分散在 Excel、聊天记录、个人日历、公司系统截图和自己的记忆里。生日、续期、回访、缴费提醒、临时待办经常不是一个入口管理，时间一久就容易漏事。

AI保顾问的当前版本先不追求大而全，而是把最基础、最高频、最容易产生业务价值的事项做好：

- 客户生日提醒
- 保单续期提醒
- 客户跟进待办
- 当天工作日历
- 完成状态记录
- Excel 导入
- 本地数据保存
- 飞书多维表格同步

## 这个项目适合谁

这个项目适合这些保险人：

- 每天需要跟进客户生日、保单续期、缴费到期和客户回访。
- 手里有客户生日表、保单表、业绩明细表或公司导出的 Excel。
- 想要一个简单的保险客户日程工具，而不是复杂的后台系统。
- 希望客户资料优先保存在自己电脑本地。
- 想用飞书多维表格做外部看板，但又不想手工维护多张表。

## 当前版本能做什么

当前开源版本重点围绕本地工作日历：

- 导入客户信息 Excel，生成客户生日提醒。
- 导入保单续期或业绩明细 Excel，根据明确缴费期间计算续期提醒。
- 续期提醒只相信明确字段，不从产品名或经验猜缴费年限。
- 支持固定年期和年龄型缴费期间。
- 缺生日、缺生效日、缴费期间无法解析、关键字段变化时进入待确认。
- 在月历里查看生日、续期和手动待办。
- 点击某一天，查看当天所有事项。
- 单条标记完成，或一键完成当天全部事项。
- 手动新增、编辑、删除客户跟进待办。
- 本地 SQLite 保存数据，并在导入前自动备份。
- 在授权后，通过本机 `lark-cli` 把脱敏客户、保单和提醒同步到飞书多维表格。

这些能力的目标不是炫技，而是让保险从业者每天少漏一个客户、少忘一次续期、多一次有效服务触达。

## 为什么强调本地优先

保险客户数据非常敏感，里面可能包含姓名、手机号、证件号、保单信息、保费金额和跟进状态。

所以当前版本采用本地优先：

- 数据默认保存在本机。
- 网页详情默认脱敏展示手机号和证件号。
- 飞书同步默认使用脱敏字段。
- 客户外部 ID 使用 hash 形态，不把姓名和证件号拼进外部 ID。
- 不提交真实客户 Excel、真实保单数据和本地 SQLite 数据库。
