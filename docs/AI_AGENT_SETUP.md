# AI Agent 安装和使用说明

这份文档给 WorkBuddy、Workbody、Codex 或其他本地 AI agent 使用。目标是让 agent 能帮助非技术用户完成安装、启动、导入检查和飞书同步。

## 工作原则

- 先保护数据，再执行操作。
- 不要把 `data/`、`.env`、Base 标识、session cookie、完整手机号、完整证件号发到聊天回复里。
- 执行写入动作前，先给用户看计划或说明影响。
- 飞书同步优先 plan，再 execute。
- 遇到授权、权限、字段不匹配时，明确说哪里卡住，不要猜测成功。

## 环境要求

- Node.js 20 或更新版本。
- npm。
- 本地端口 `3001` 和 `4173` 可用。
- 如需飞书同步：本机已安装并授权 `lark-cli`。

检查命令：

```bash
node -v
npm -v
```

飞书检查：

```bash
lark-cli auth status
```

如果 `lark-cli` 不存在或未授权，请先协助用户安装并完成飞书登录授权，再继续飞书同步。

## 首次安装

```bash
npm install
```

基础验证：

```bash
npm run typecheck
npm test
npm run build
```

## 启动本地工具

```bash
npm run start
```

启动后浏览器访问：

```text
http://127.0.0.1:4173/
```

API 默认地址：

```text
http://127.0.0.1:3001
```

如果端口被占用，先告诉用户哪个端口被占用，并让用户关闭已有进程。

## 可选访问密码

用户可以这样启动：

```bash
CUSTOMER_REMINDERS_PASSWORD="用户自定密码" npm run start
```

这个密码只保护网页入口。数据库加密需要用户在维护页单独启用数据密码。不要在回复里重复用户密码。

## 数据加密和云备份

维护页可以启用本地 SQLite 数据库加密。已加密数据库启动后需要先输入数据密码解锁。修改数据密码时必须输入当前数据密码；当前密码错误不会改密。

坚果云云备份只作为额外备份出口，不是多设备实时同步。数据库未加密时，不允许连接或上传云备份。坚果云 WebDAV 需要第三方应用密码，不能使用官网登录密码或验证码。

协助用户配置坚果云时，提醒用户按这个流程获取应用密码：

1. 登录坚果云官网。
2. 进入账户信息，打开安全选项。
3. 找到第三方应用管理或应用密码。
4. 新建应用密码，名称可填 `客户提醒备份`。
5. 复制生成出来的那串密码。
6. 回到本工具，账号填坚果云邮箱，密码填刚生成的应用密码。

不要在聊天回复、日志、测试快照或文档里保存用户的坚果云账号、应用密码、数据库密码或验证码。

## 导入说明

用户需要准备两类 Excel：

- 客户信息表：包含客户姓名、证件号、出生日期、手机号等字段。
- 保单续期/业绩明细表：包含投保人、被保人、保单号码、保险产品、保费、缴费期间、生效时间等字段。

字段说明见：

```text
docs/sample-data.md
```

建议导入顺序：

1. 先启动本地工具。
2. 在网页维护页上传客户信息表和保单续期表。
3. 先做预检，确认客户、保单、提醒、待确认数量。
4. 用户确认后正式导入。
5. 导入后处理待确认事项，再回到工作日历检查当天和本月提醒。

## 飞书同步

飞书同步详细步骤见：

```text
docs/agent-feishu-sync.md
```

用户需要提供一个可编辑的飞书多维表格链接，例如：

```text
https://xxx.feishu.cn/base/xxxx
```

同步前必须确认：

- 链接是飞书多维表格，不是普通文档或电子表格。
- 当前飞书账号有编辑权限。
- 本机 `lark-cli auth status` 已授权。
- 本地导入和待确认已经处理到用户认可的状态。

## 回报模板

完成后只回报业务结果：

```text
本地工具已启动，可以打开 http://127.0.0.1:4173/。
本次导入后：客户 x，保单 y，提醒 z，待确认 n。
飞书同步计划/执行：客户 x，保单 y，提醒 z。
续期提醒只包含未来 60 天内要跟进的事项；更远续期在保单表看“下次续期”和“缴费结束年”。
手机号和证件号在飞书侧保持脱敏。
下一步需要用户处理：……
```

不要展示完整手机号、证件号、Base 标识、session cookie 或数据库路径里的敏感内容。
